## Weather Identification System ##
---
## Introduction

The Weather Identification System is a full-stack web application developed to provide real-time weather information for any city. The system allows users to enter a city name and view the current weather conditions such as temperature, humidity, wind speed, and weather status.


## Objective

The main objective of this project is to:

Develop a simple full-stack application using frontend and backend technologies
Fetch real-time weather data from an external API
Display weather information in a user-friendly interface


## Technologies Used
Frontend: HTML, CSS, JavaScript
Backend: Node.js, Express.js
API: wttr.in (Weather API without authentication)
Tool: Visual Studio Code


## System Architecture

The system follows a client-server architecture:

The client (frontend) collects user input (city name)
The server (backend) processes the request
The backend fetches data from the weather API
The processed data is sent back to the frontend and displayed


## Working Principle
User enters a city name in the input field
The request is sent to the backend server
The backend calls the weather API
Weather data is retrieved and processed
The result is displayed on a new page


## Features
Real-time weather data
Multi-page navigation
Displays temperature, humidity, wind speed, and weather condition
Simple and user-friendly interface


## Advantages
Easy to use
No API key required
Fast response
Suitable for beginners


## Limitations
Requires internet connection
Limited UI design
Depends on third-party API


## Future Enhancements
Add 5-day weather forecast
Improve UI design with CSS
Add location-based weather detection
Convert into mobile application

## Conclusion

The Weather Identification System successfully demonstrates the implementation of a full-stack web application. It helps in understanding how frontend and backend technologies interact with external APIs to provide real-time data to users.
